# 工作汇报场景指南

> 适用：季度汇报、项目汇报、述职、工作总结

---

## 场景特征

| 特征 | 说明 |
|------|------|
| **PPT等级** | 二等 - 逻辑层 |
| **信息密度** | 中（结论先行） |
| **观众特点** | 时间有限、关注成果 |
| **核心目标** | 展示价值、获得认可 |

---

## 核心设计原则

### ✅ 应该做的

| 原则 | 说明 | 示例 |
|------|------|----------|
| 结论先行 | 领导先知道结果 | 标题即结论 |
| 成果导向 | 展示功劳和价值 | 用数据证明 |
| 结构清晰 | 按模块/时间分类 | 成果+问题+计划 |
| 数据支撑 | 用数字说话 | 增长率、完成任务数 |
| 问题透明 | 不回避+有方案 | 问题+解决方案 |

### ❌ 不不应该做的

| 错误 | 说明 |
|------|------|
| 流水账 | 没有重点 |
| 只有苦劳 | 没有功劳（价值） |
| 回避问题 | 不够透明 |
| 太谦虚 | 价值没体现 |
| 太长 | 领导没耐心 |

## 与乔布斯式的区别

| 工作汇报 | 乔布斯式发布 |
|----------|--------------|
| 展示成果 | 打动人心 |
| 逻辑清晰 | 制造震撼 |
| 可以有数据表格 | 极简图表 |
| 结论先行 | 悬念开场 |
| 结构化 | 故事线 |

## 黄金结构

```
第1页：封面（标题 + 季度/项目名）
第2页：核心成果（最重要的一件事）
第3页：关键数据（用图表展示）
第4页：重点项目1（成果+贡献）
第5页：重点项目2（成果+贡献）
第6页：遇到的问题 + 解决方案
第7页：下阶段计划
第8页：需要的支持/资源
```

**简化版（时间紧时）**：
```
第1页：核心成果一句话
第2页：关键数据
第3页：主要成果（2-3个）
第4页：下阶段计划
```

## 数据展示技巧

| 数据类型 | 展示方式 | 效果 |
|----------|----------|------|
| 增长 | 折线图/柱状图 | 直观 |
| 完成率 | 饼图/环形图 | 清晰 |
| 排名 | 条形图 | 一目了然 |
| 对比 | 并列柱状图 | 直观 |

---

## 检查清单

```
□ 标题是否直接说明核心成果？
□ 最重要的成果是否在前面（结论先行）？
□ 是否有数据支撑（不是自说自话）？
□ 遇到的问题是否坦诚+有方案？
□ 下阶段计划是否清晰可执行？
□ 是否主动提出需要什么支持？
□ 页数是否控制在10页以内？
□ 数据是否有来源/依据？
□ 整体是否结构清晰（领导能跟上）？
□ 是否有适当的视觉元素（不是纯文字）？
```
