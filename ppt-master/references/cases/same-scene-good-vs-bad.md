# 同一场景好PPT vs 差PPT 对比

> 同一个场景下，好的PPT和差的PPT差距在哪里？通过直接对比，一目了然。

---

## 场景1：年度工作汇报

### 场景背景

| 项目 | 内容 |
|------|------|
| 场景 | 年度工作总结汇报 |
| 受众 | 部门领导/公司高层 |
| 时长 | 10-15分钟 |
| 目的 | 展示成绩、争取资源 |

---

### 差PPT的典型问题

| 问题类型 | 具体表现 |
|---------|---------|
| 封面恐惧症 | "XX公司2024年度工作总结与2025年工作计划——综合管理部" |
| 数据堆砌症 | 20个指标的表格，每个都有同比环比 |
| 文字海啸 | 每页200+字，照着念要1小时 |
| 没有记忆点 | 讲完就忘，不知道重点在哪 |

---

### 好PPT vs 差PPT 对比

| 维度 | 差PPT | 好PPT | 好在哪 |
|------|-------|-------|--------|
| **封面** | 部门名+日期+文号 | "这一年，我们让成本降低了20%" | 有记忆点，能引发好奇 |
| **成绩页** | 20个数据指标 | "3个数字说明一切"：收入X亿+成本降X%+客户增X% | 聚焦，观众能记住 |
| **做法页** | 详细文字描述5种做法 | 一张图展示关键举措 | 直观，一眼看懂 |
| **数据页** | 密密麻麻的表格 | 一张趋势图+一句话解读 | 趋势比数字更重要 |
| **结尾页** | "谢谢大家" | "明年目标：X亿，需要支持：Y" | 有行动号召 |

---

### 关键差异总结

```
差PPT：罗列事实
好PPT：讲故事

差PPT：我做了什么
好PPT：带来了什么价值

差PPT：面面俱到
好PPT：只说最重要的
```

## 场景2：产品发布会/路演

### 场景背景

| 项目 | 内容 |
|------|------|
| 场景 | 新产品发布/商业路演 |
| 受众 | 投资人/客户/合作伙伴 |
| 时长 | 15-20分钟 |
| 目的 | 说服、融资、建立合作 |

---

### 差PPT的典型问题

| 问题类型 | 具体表现 |
|---------|---------|
| 技术自嗨 | 满屏技术术语，看不懂 |
| 业务分散 | 什么都做，什么都说不清 |
| 市场规模虚 | "万亿市场蓝海"跟我有什么关系 |
| 情怀泛滥 | 只有愿景，没有商业模式 |
| 团队缺失 | 只有名字，没有背景 |

---

### 好PPT vs 差PPT 对比

| 维度 | 差PPT | 好PPT | 好在哪 |
|------|-------|-------|--------|
| **封面** | "基于XXX技术的下一代YYY平台" | "让XXX成本降低50%" | 一句话说明价值 |
| **问题页** | 复杂的技术原理图 | "你有没有遇到过X问题" | 引发共鸣 |
| **方案页** | 技术架构图 | 一张使用场景图+一句话说明 | 看得懂 |
| **数据页** | 20个指标 | 3个核心数字：客户数、增长率、NPS | 聚焦 |
| **对比页** | 表格对比10个指标 | 一张效果图对比 | 直观 |
| **团队页** | 名字+职位 | 照片+一句话成就 | 让人相信 |
| **融资页** | "融资5000万" | "融资5000万，用于XXX，预计YYY" | 让人相信能成功 |

---

### 关键差异总结

```
差PPT：我是谁
好PPT：能为你解决什么问题

差PPT：技术多先进
好PPT：客户用完多开心

差PPT：市场多大
好PPT：你能拿到多少

差PPT：团队多豪华
好PPT：他们做出过什么成绩
```

## 场景3：培训课件

### 场景背景

| 项目 | 内容 |
|------|------|
| 场景 | 企业内部培训 |
| 受众 | 员工/新人 |
| 时长 | 30-60分钟 |
| 目的 | 知识传递、技能培训 |

---

### 差PPT的典型问题

| 问题类型 | 具体表现 |
|---------|---------|
| 文字海啸 | 概念+定义+原则+注意事项，5条以上 |
| 缺乏互动 | 纯念稿，没有互动设计 |
| 没有案例 | 只有干巴巴的理论 |
| 节奏平淡 | 从头到尾一个节奏 |

---

### 好PPT vs 差PPT 对比

| 维度 | 差PPT | 好PPT | 好在哪 |
|------|-------|-------|--------|
| **封面** | "XXX培训课程——员工发展部" | "3分钟学会XXX" | 有目标感 |
| **内容页** | 5个知识点+定义 | 1个核心概念+1个案例 | 记得住 |
| **案例页** | "例如：XXX案例" | 真实故事+前后对比 | 有共鸣 |
| **互动页** | 无 | 提问+投票+小组讨论 | 参与感 |
| **总结页** | "今天学了X个知识点" | "记住一件事：XXX" | 带走明确信息 |

---

### 关键差异总结

```
差PPT：老师讲什么
好PPT：学生能记住什么

差PPT：知识点罗列
好PPT：一个问题+一个解决方案

差PPT：讲完就走
好PPT：留下一个记忆点
```

## 场景4：学术汇报/论文答辩

### 场景背景

| 项目 | 内容 |
|------|------|
| 场景 | 学术答辩/研究汇报 |
| 受众 | 评审专家/导师 |
| 时长 | 15-20分钟 |
| 目的 | 通过评审、获得认可 |

---

### 差PPT的典型问题

| 问题类型 | 具体表现 |
|---------|---------|
| 文字海啸 | 论文直接复制，公式堆砌 |
| 没有图表 | 只有文字描述，没有可视化 |
| 节奏失控 | 10分钟讲不完第一部分 |
| 缺乏重点 | 什么都想讲，什么都讲不透 |

---

### 好PPT vs 差PPT 对比

| 维度 | 差PPT | 好PPT | 好在哪 |
|------|-------|-------|--------|
| **封面** | 论文题目+作者+学校+导师 | "XXX问题，我找到了答案" | 有核心观点 |
| **研究背景** | 10篇文献综述 | "这个问题，学术界有3种观点，我是第4种" | 有立场 |
| **方法页** | 复杂公式+流程图 | 一句话方法名+一张技术路线图 | 讲得清 |
| **数据页** | 10个实验表格 | 1个核心发现图+1句话总结 | 说得清 |
| **结论页** | "本研究证明了XXX" | "这个发现，意味着YYY" | 有意义 |

---

### 关键差异总结

```
差PPT：我做了什么研究
好PPT：我发现了什么

差PPT：方法多复杂
好PPT：方法解决了什么问题

差PPT：数据多详细
好PPT：结论多明确
```

## 场景5：客户提案

### 场景背景

| 项目 | 内容 |
|------|------|
| 场景 | 投标提案/客户汇报 |
| 受众 | 甲方客户 |
| 时长 | 20-30分钟 |
| 目的 | 赢得项目、建立信任 |

---

### 差PPT的典型问题

| 问题类型 | 具体表现 |
|---------|---------|
| 自说自话 | 只讲自己想讲的，不关心客户需求 |
| 方案堆砌 | 10个解决方案，客户不知道选哪个 |
| 缺少案例 | 只讲方案，没有证明 |
| 价格模糊 | 不提前说清楚，可能超预算 |

---

### 好PPT vs 差PPT 对比

| 维度 | 差PPT | 好PPT | 好在哪 |
|------|-------|-------|--------|
| **封面** | 公司介绍+业务范围 | "针对你们的XXX问题，我们有方案" | 关心客户 |
| **需求理解** | "我们理解你们的需求是......" | 直接复述客户原话+确认 | 证明倾听了 |
| **方案页** | 10个解决方案 | 1个推荐方案+1个备选 | 有主次 |
| **案例页** | 公司介绍PPT | 同行业成功案例+数据 | 有证明 |
| **团队页** | 组织架构图 | 实际参与人员+他们的经验 | 让人放心 |
| **价格页** | "根据需求......" | "总价X，含以下内容......" | 清晰 |

---

### 关键差异总结

```
差PPT：我是谁
好PPT：我理解你的问题

差PPT：我能做什么
好PPT：我为和你一样的问题做过什么

差PPT：价格面议
好PPT：明码标价
```

## 对比学习法

### 如何使用这些对比

```
1. 确定场景
   → 找到对应的场景对比

2. 逐项对比
   → 差PPT为什么差？
   → 好PPT为什么好？

3. 自查
   → 我的PPT更接近哪个？
   → 我容易犯哪个错误？

4. 改进
   → 差的地方，参考好PPT改
   → 好的地方，保持并强化
```

### 一句话总结

> **好的PPT不是"我有多厉害"，而是"你能获得什么"。**
