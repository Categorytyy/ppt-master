# PPT设计思维指南

> 本章源自《成为PPT高手》（马馺）等经典文献，系统阐述PPT设计的核心思维体系。

---

## 1. 五种核心思维

**来源**：《成为PPT高手：从思维开始重塑PPT能力》- 马馺

PPT高手与普通选手的本质区别在于思维方式。本章揭示五种让PPT从"技术活"升华为"思维活"的核心思维框架。

### 1.1 关系思维

**核心**：以目标对象为核心的思维方式，找到观众与PPT之间的关系。

**应用方法**

| 问题 | 回答方向 |
|------|----------|
| 我的观众是谁？ | 年龄/职业/职位/背景 |
| 他们的关注点是什么？ | 利益/痛点/兴趣 |
| 我想让他们记住什么？ | 核心信息 |
| 他们已经知道什么？ | 避免重复 |

**实战案例**

京东金融发布会：团队通过调研发现"小白卡"目标用户是年轻群体，于是围绕"如何让年轻人记住这款为他们打造的信用卡"设计PPT，用情景化案例代替功能罗列，效果显著。

**核心观点**
> "做给谁看，谁就是你的观众。" —— 马馺

---

### 1.2 预期思维

**核心**：以结果导向，打造演示场景。

**应用方法**

| 问题 | 回答方向 |
|------|----------|
| 演示结束后，观众应该做什么？ | 行动目标 |
| 演示场景是什么？ | 会议室/发布会/教室 |
| 预期效果是什么？ | 说服/告知/培训 |

**实战案例**

华为Mate20伦敦发布会 vs 上海发布会：
- 伦敦市场不熟悉华为，PPT用5-6页回忆技术历史，先树IP再推产品
- 上海市场熟悉华为，PPT着重展示外媒评价，用口碑背书

**核心观点**
> "不同的演示场景，需要不同的PPT逻辑。" —— 马馺

---

### 1.3 简化思维

**核心**：化繁为简，高效传递。核心信息只有一件。

**应用方法**

| 步骤 | 操作 |
|------|------|
| 第一步 | 找出最重要的信息（只有1件） |
| 第二步 | 删除可有可无的内容 |
| 第三步 | 用最简洁的方式呈现 |

**检查清单**

```
□ 这页的核心信息是什么？
□ 除了核心信息，还有什么可以删掉？
□ 能否用一句话概括这一页？
```

**核心观点**
> "你没有办法让一张塞满内容的幻灯片变得好看，但你可以轻易让一张简洁的幻灯片变得充实。" —— Garr Reynolds

---

### 1.4 传播思维

**核心**：制造记忆点，强化传播效益。

**记忆点类型**

| 类型 | 例子 | 适用场景 |
|------|------|----------|
| 金句 | "DT时代，数据就是石油" | 核心观点输出 |
| 画面 | 数据可视化"数字树" | 视觉冲击 |
| 故事 | 创始人逆袭故事 | 品牌/产品介绍 |
| 对比 | "Before vs After" | 解决方案展示 |

**应用方法**

| 步骤 | 操作 |
|------|------|
| 第一步 | 确定要传播的核心信息 |
| 第二步 | 设计一个让人记住的"记忆点" |
| 第三步 | 确保这个记忆点容易被复述 |

**核心观点**
> "让人记住什么，决定了PPT的传播价值。" —— 马馺

---

### 1.5 设计思维

**核心**：从结构到色彩再到字体排版，让PPT成为最佳作品。

**设计四要素**

| 要素 | 核心问题 | 检查点 |
|------|----------|--------|
| 结构 | 逻辑是否清晰？ | 是否有清晰的故事线？ |
| 色彩 | 配色是否专业？ | 是否不超过3种主色？ |
| 字体 | 字体是否合适？ | 是否区分标题和正文？ |
| 排版 | 布局是否有序？ | 是否有对齐和留白？ |

**应用方法**

| 步骤 | 操作 |
|------|------|
| 第一步 | 先确定内容结构（逻辑层） |
| 第二步 | 再设计视觉呈现（视觉层） |
| 第三步 | 最后检查细节（执行层） |

**核心观点**
> "设计思维让内容如虎添翼。" —— 马馺

## 2. KANO需求模型应用

**来源**：《成为PPT高手》- 马馺（产品经理工具）

### 2.1 模型解释

KANO模型将用户需求分为五个层级，在PPT设计中的应用：

```
┌─────────────────────────────────────────────────────────┐
│                    KANO模型在PPT中的应用                  │
├─────────────────────────────────────────────────────────┤
│                                                          │
│  兴奋型需求 → 超出预期的惊喜（做了会有"哇"的效果）          │
│     ↑                                                    │
│  期望型需求 → 做到越好越满意（做了会有"不错"的效果）        │
│     ↑                                                    │
│  基本型需求 → 必须做到的基本要求（不做会"不满"）            │
│                                                          │
└─────────────────────────────────────────────────────────┘
```

### 2.2 需求层级详解

| 需求类型 | PPT表现 | 例子 |
|----------|---------|------|
| **基本型需求** | 必须做到的基本要求 | 结构清晰、配色专业、无错别字、逻辑通顺 |
| **期望型需求** | 做到越好越满意 | 精美排版、创意图表、流畅动画、吸睛开场 |
| **兴奋型需求** | 超出预期的惊喜 | 独特动画、记忆金句、动人故事、意外数据 |

### 2.3 实战应用

**制作前：分析观众想要什么**

| 问题 | 分析方向 |
|------|----------|
| 基本型需求 | 观众期望看到什么？（结构清晰、重点突出） |
| 期望型需求 | 观众希望得到什么额外价值？（专业感、信任感） |
| 兴奋型需求 | 什么会让观众"哇"？（独特见解、震撼数据） |

**制作中：分配设计资源**

| 需求类型 | 投入比例 | 说明 |
|----------|----------|------|
| 基本型需求 | 70% | 确保不出错 |
| 期望型需求 | 25% | 做得更好 |
| 兴奋型需求 | 5% | 适当加入惊喜 |

**核心观点**
> "满足了基本型需求，观众不会满意但也不会不满；满足了期望型需求，观众会满意；满足了兴奋型需求，观众会'哇'。" —— 马馺

## 3. 观众分级理论

**来源**：《成为PPT高手》- 马馺

### 3.1 观众分层

| 观众层级 | 比例 | 策略 | 内容分配 |
|----------|------|------|----------|
| **核心观众** | 70% | 重点满足他们的需求 | 70%内容针对他们 |
| **重要观众** | 20% | 适当照顾 | 20%内容涉及 |
| **外围观众** | 10% | 兼顾但不重点 | 10%内容提及 |

### 3.2 如何识别核心观众

| 问题 | 回答 |
|------|------|
| 谁最可能做决定？ | 那就是核心观众 |
| 谁最关心这个主题？ | 那就是核心观众 |
| 如果只能满足一个人，是谁？ | 那就是核心观众 |

### 3.3 实战案例

**业绩汇报场景分析**

| 观众 | 重要性 | 关注点 | 内容策略 |
|------|---------|--------|----------|
| 公司老板 | ⭐⭐⭐⭐⭐ | 部门价值/业绩/成果 | 70%内容：成果总结+价值贡献 |
| 部门领导 | ⭐⭐⭐⭐ | 领导力/团队/过程 | 20%内容：曲折故事+团队亮点 |
| 跨部门同事 | ⭐⭐⭐ | 资源支持/协作 | 10%内容：致谢+协作说明 |
| 同部门同事 | ⭐⭐ | 执行/配合 | 10%内容：感谢 |

### 3.4 内容分配原则

```
核心观众（70%）：
- 他们最想听什么？
- 什么能打动他们？
- 什么能让他们做决定？

重要观众（20%）：
- 他们的关注点是什么？
- 如何让他们感到被重视？

外围观众（10%）：
- 兼顾即可
- 不需要深入
```

**核心观点**
> "给听众分级，按层级满足需求。" —— 马馺

## 4. 三等PPT论

**来源**：《成为PPT高手》- 马馺

### 4.1 等级划分

| 等级 | 评判标准 | 特征 | 区分度 |
|------|----------|------|--------|
| **三等PPT** | 看技能 | 会用软件，能做出来 | 操作技巧 |
| **二等PPT** | 看逻辑 | 逻辑清晰，内容组织合理 | 结构思维 |
| **一等PPT** | 看思维 | 思维领先，能打动人 | 思想深度 |

### 4.2 各等级详解

**三等PPT：技能层**

- 熟练使用软件功能
- 会使用快捷键
- 会添加动画和切换效果
- 关注点："怎么做"

**二等PPT：逻辑层**

- 有清晰的逻辑结构
- 内容组织有条理
- 信息分层合理
- 关注点："说什么"

**一等PPT：思维层**

- 有独到的见解和思想
- 能打动人、说服人
- 有故事、有情感
- 关注点："为什么"

### 4.3 提升路径

```
技能 → 逻辑 → 思维

1. 先掌握基本操作（三等）
   ↓
2. 学会组织内容逻辑（二等）
   ↓
3. 最终形成独特思维（一等）
```

### 4.4 自检问题

| 问题 | 对应等级 |
|------|----------|
| 我的PPT操作熟练吗？ | 三等 |
| 我的PPT逻辑清晰吗？ | 二等 |
| 我的PPT能打动人吗？ | 一等 |

**核心观点**
> "三等PPT看技能，二等PPT看逻辑，一等PPT看思维。" —— 马馺

## 5. 五问法扩展

**来源**：《成为PPT高手》- 马馺

### 5.1 经典5W1H

| 问题 | 核心追问 |
|------|----------|
| **What** | 核心信息只有一件，是什么？ |
| **Who** | 核心观众是谁？他们想要什么？ |
| **Where** | 演示场景适合什么风格？ |
| **When** | 时长决定内容密度 |
| **Why** | 观众为什么要听？ |
| **How** | 如何让观众记住？ |

### 5.2 新增两问

| 问题 | 核心追问 |
|------|----------|
| **Pages** | 预计多少页？（建议10-30页） |
| **Impact** | 演示结束后，观众会做什么？ |

### 5.3 Impact问题详解

**为什么Impact最重要？**

| 问题 | 回答 |
|------|------|
| 演示的目的是什么？ | 让观众行动 |
| 如何衡量成功？ | 观众做了什么 |
| PPT只是工具，行动才是目的 | 结束后的变化 |

**实战应用**

| 场景 | Impact问题 | 预期行动 |
|------|------------|----------|
| 商业提案 | 客户会签约吗？ | 签署合同 |
| 产品发布 | 观众会购买吗？ | 下单购买 |
| 培训演讲 | 学员会应用吗？ | 实践操作 |
| 学术汇报 | 评审会通过吗？ | 给予通过 |

### 5.4 完整五问法检查清单

```
□ What：核心信息是什么？能否用一句话概括？
□ Who：核心观众是谁？他们的关注点是什么？
□ Where：演示场景是什么？适合什么风格？
□ When：时长是多少？内容密度如何控制？
□ Why：观众为什么要听？他们的收益是什么？
□ How：如何让观众记住？记忆点是什么？
□ Pages：预计多少页？内容分配合理吗？
□ Impact：演示结束后，观众会做什么？
```

**核心观点**
> "PPT的最终目的是让观众行动。" —— 马馺

## 6. 设计思维应用流程

### 6.1 需求分析阶段

```
收到PPT需求 →
  ↓
用五种思维审视
  ├─ 关系思维：观众是谁？
  ├─ 预期思维：目标是什么？
  ├─ 简化思维：核心信息是什么？
  ├─ 传播思维：记忆点是什么？
  └─ 设计思维：如何设计？
  ↓
回答五问法
  ↓
确定PPT框架
```

### 6.2 内容策划阶段

```
确定内容 →
  ↓
用KANO模型分配
  ├─ 基本型需求（70%）：必须说的
  ├─ 期望型需求（25%）：加分项
  └─ 兴奋型需求（5%）：惊喜点
  ↓
用观众分级分配
  ├─ 核心观众70%内容
  ├─ 重要观众20%内容
  └─ 外围观众10%内容
```

### 6.3 视觉设计阶段

```
内容确定 →
  ↓
结构设计（逻辑层）
  ├─ 故事线是否清晰？
  ├─ 信息分层是否合理？
  └─ 逻辑是否通顺？
  ↓
视觉设计（视觉层）
  ├─ 配色是否专业？
  ├─ 字体是否合适？
  ├─ 排版是否有序？
  └─ 是否有足够留白？
```

### 6.4 三等PPT自检

```
□ 三等（技能）：操作是否熟练？基本功能是否掌握？
□ 二等（逻辑）：结构是否清晰？信息是否分层？
□ 一等（思维）：是否能打动人？是否有独特见解？
```

## 7. 核心思维框架总结

### 7.1 思维工具箱

| 工具 | 适用阶段 | 核心问题 |
|------|----------|----------|
| 五种核心思维 | 需求分析 | 如何全面审视PPT？ |
| KANO模型 | 内容策划 | 内容如何分配优先级？ |
| 观众分级 | 内容分配 | 内容如何针对不同观众？ |
| 三等PPT论 | 自检 | 我的PPT是哪个等级？ |
| 五问法 | 需求确认 | 八个问题是否都回答了？ |

### 7.2 应用顺序

```
第一步：需求分析 → 五种思维 + 五问法
         ↓
第二步：内容策划 → KANO模型
         ↓
第三步：内容分配 → 观众分级
         ↓
第四步：视觉设计 → 设计思维
         ↓
第五步：自检 → 三等PPT论
```

### 7.3 核心理念

> "做PPT不是做文档，而是帮助演讲者与观众建立沟通。"

> "技术可以速成，思维需要积累。"

> "一等PPT靠思想打动人，二等PPT靠逻辑说服人，三等PPT靠技巧完成人。"

## 8. 认知偏见与PPT设计

**来源**：《Universal Principles of Design》

> 认知偏见是人类视觉和决策系统中的系统性偏差。了解它们，可以帮助PPT设计"顺应"人类认知规律，而不是"对抗"。

### 8.1 设计相关认知偏见速查

| 偏见名称 | 核心洞察 | PPT应用 |
|---------|---------|---------|
| **美观-可用性效应** | 美观的设计被认为更好用 | 投资设计细节 |
| **轮廓偏见** | 人眼优先感知轮廓 | 用描边/色块创造边界 |
| **图形-背景偏见** | 视觉自动分离主体与背景 | 清晰的层次结构 |
| **冯·雷斯托夫效应** | 与众不同更容易记忆 | 核心信息要"跳出来" |
| **框架效应** | 表述方式影响决策 | 用正向框架表述 |
| **系列位置效应** | 开头和结尾比中间更容易记忆 | 重点放开头和结尾 |

### 8.2 美观-可用性效应详解

**原理**：人们倾向于认为美观的设计更易用、更有价值。

**PPT应用**：
| 做法 | 效果 |
|------|------|
| 花时间优化配色 | 观众认为内容更有价值 |
| 保持排版专业 | 观众更容忍小问题 |
| 风格统一 | 增强可信度 |

### 8.3 冯·雷斯托夫效应详解

**原理**：在多个相似元素中，与众不同的那个最容易记忆。

**PPT应用**：
```
核心记忆点设计 = 独特颜色 + 独特尺寸 + 独特位置

❌ 错误：所有内容都是蓝色14pt
✓ 正确：核心数据用橙色48pt，置于视觉焦点
```

### 8.4 框架效应详解

**原理**：同样的事实，不同的表述方式产生不同反应。

**PPT应用**：

| 负向框架（避免） | 正向框架（推荐） |
|----------------|----------------|
| "不这样做会损失" | "这样做会获得" |
| "失败率30%" | "成功率70%" |
| "我们的问题是什么" | "我们的机会是什么" |

### 8.5 系列位置效应详解

**原理**：系列中的开头和结尾比中间更容易记忆。

**PPT应用**：
| 位置 | 适合放什么 |
|------|----------|
| 封面/开场 | 核心观点、演讲主题 |
| 结尾 | 行动号召、核心金句 |
| 中间 | 支撑论据（需要强化记忆点设计） |
