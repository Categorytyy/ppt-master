# 学术汇报场景指南

> 适用：论文答辩、学术会议、研究分享、学术演讲

---

## 场景特征

| 特征 | 说明 |
|------|------|
| **PPT等级** | 二等 - 逻辑层 |
| **信息密度** | 中高（可多要点） |
| **观众特点** | 专业背景强、会拿PPT当资料 |
| **核心目标** | 展示研究价值、获得认可 |

---

## 核心设计原则

### ✅ 应该做的

| 原则 | 说明 | 示例 |
|------|------|------|
| 专业严谨 | 配色、字体、结构都要专业 | 学术蓝+白、清晰分层 |
| 逻辑清晰 | 让评审快速理解研究脉络 | 时间线/流程图 |
| 信息完整 | 可以多要点（不同于乔布斯） | 每页2-4个相关要点 |
| 数据支撑 | 图表+数据+来源 | 柱状图、折线图 |
| 留参考文献 | 学术规范 | 角落小字 |

### ❌ 不应该做的

| 错误 | 说明 |
|------|------|
| 过于极简 | 学术汇报需要信息密度 |
| 堆砌文字 | 还是要有逻辑结构 |
| 缺少数据 | 空口无凭不行 |
| 没有引用 | 缺乏可信度 |

---

## 信息密度指南

**与乔布斯极简法的区别**：

| 乔布斯式发布会 | 学术汇报 |
|----------------|----------|
| 一页一信息 | 可以一页2-4个**相关**要点 |
| 极度留白 | 适当紧凑但要有结构 |
| 零文字 | 可以有说明文字 |
| 纯视觉 | 需要数据图表支撑 |

**学术汇报正确姿势**：
- 每页一个**主题**
- 每个主题可以有2-4个**相关要点**
- 要点之间要有**逻辑关系**（不是简单罗列）
- 用**图表/数据**代替文字描述

## 黄金结构

```
第1页：封面（标题 + 作者 + 单位）
第2页：目录（研究背景 → 方法 → 结果 → 结论）
第3页：研究背景（问题提出）
第4页：研究方法
第5-7页：研究结果（数据+图表）
第8页：讨论/结论
第9页：参考文献
第10页：致谢
```

---

## 检查清单

```
□ 标题是否专业、包含关键词？
□ 研究问题是否清晰？
□ 方法论是否可信？
□ 数据是否有来源标注？
□ 图表是否清晰可读？
□ 逻辑是否清晰（评审能跟上）？
□ 是否有适当的留白（不是太满）？
□ 是否符合学术规范（引用、致谢）？
□ 字体是否统一（标题vs正文）？
□ 配色是否专业（不是太花哨）？
```

## 与其他场景的对比

| 原则 | 学术汇报 | 产品发布 |
|------|----------|----------|
| 信息密度 | 中高 | 极低 |
| 一页要点 | 2-4个相关 | 1个 |
| 数据展示 | 必须 | 可选 |
| 留白程度 | 适中 | 大量 |
| 乔布斯极简法 | ❌ 不适用 | ✅ 必须 |
