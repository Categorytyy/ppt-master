# 商业发布场景指南

> 适用：产品发布、路演、乔布斯式发布会、品牌展示

---

## 场景特征

| 特征 | 说明 |
|------|------|
| **PPT等级** | 一等 - 思维层 |
| **信息密度** | **极低**（一页一信息） |
| **观众特点** | 时间有限、需要被震撼 |
| **核心目标** | 打动人、让人记住、产生行动 |

---

## 核心设计原则

### ✅ 乔布斯极简法（必须遵守）

| 原则 | 说明 | 示例 |
|------|------|----------|
| 一页一信息 | 每页只说一件事 | iPhone发布：一页只展示一个功能 |
| 大数字 | 关键数据超级放大 | "最薄的 MacBook Air" |
| 图片优先 | 产品图就是全部 | 乔布斯从信封里抽出 MBA |
| 故事线 | 每一页都在讲故事 | 从问题→解决方案→产品 |
| 制造惊喜 | 让观众"哇" | 从信封里抽出产品 |

### ❌ 不应该做的

| 错误 | 说明 |
|------|------|
| 信息堆砌 | 一页放太多内容 |
| 照着念 | 演讲变成朗读 |
| 没有记忆点 | 观众记不住 |
| 缺乏故事 | 枯燥无味 |

---

## 与学术汇报的本质区别

| 乔布斯式发布 | 学术汇报 |
|--------------|----------|
| 打动人 | 说明白 |
| 情绪共鸣 | 逻辑严密 |
| 极简（一页一信息） | 信息密度高（多要点） |
| 视觉冲击 | 数据支撑 |
| 不需要参考文献 | 必须引用 |
| PPT是主角 | 演讲者是主角 |

## 版式风格

**推荐风格**：Rounded / Pill

| 元素 | 数值 |
|------|------|
| 圆角 | 0.15" ~ 0.3" |
| 页面边距 | 0.5" ~ 0.6" |
| 元素间距 | 0.25" ~ 0.4" |
| 留白比例 | 60-70% |

## 黄金结构

```
第1页：封面（悬念/大主题/品牌）
第2页：痛点（用户面临的问题）
第3页：解决方案（我们的产品）
第4页：产品展示（震撼大图）
第5页：核心功能1（聚焦）
第6页：核心功能2（聚焦）
第7页：核心功能3（聚焦）
第8页：数据/证明（第三方背书）
第9页：价格 + 购买方式
第10页：结束页（品牌/CTA）
```

---

## 乔布斯式发布技巧

| 技巧 | 示例 | 效果 |
|------|------|------|
| 从信封抽出 | MBA发布 | 制造惊喜 |
| 大数字 | "1.8cm"（厚度） | 建立里程碑 |
| 对比 | "vs 竞品" | 直观展示优势 |
| 重复 | 核心词多次出现 | 加深记忆 |
| 悬念 | 先不讲产品 | 制造好奇 |

---

## 检查清单

```
□ 是否有悬念/惊喜开场？
□ 是否一页只说一件事？
□ 是否有大数字/震撼画面？
□ 是否有故事线（问题→方案→产品）？
□ 观众能在3秒内理解本页吗？
□ 是否有记忆点（金句/画面/数字）？
□ 是否有第三方背书？
□ 价格和购买方式是否清晰？
□ 整体是否简洁（不是堆砌）？
□ 页数是否控制在10页以内？
```

## 视觉化技巧

**"图 > 文"原则**：

| 要表达 | 不要 | 要 |
|--------|------|-----|
| 增长 | "我们增长了80%" | 一个陡峭的上升曲线 |
| 轻薄 | "只有1.8cm" | 一张与信封对比的图 |
| 用户量 | "1亿用户" | 一个巨大的数字 |
| 对比 | "比竞品快3倍" | 并排对比图 |
