# 图表设计实战指南

## 图表选择原则

### 选择图表的核心逻辑

```markdown
选择图表之前，先问自己：想展示什么关系？

比较 → 柱形图、条形图
趋势 → 折线图
比例 → 饼图、环形图
分布 → 散点图、直方图
关联 → 气泡图
流程 → 流程图、时间轴
```

### 数据关系与图表匹配

| 数据关系 | 推荐图表 | 禁用场景 |
|----------|----------|----------|
| 类别比较 | 柱形图、条形图 | 超过7个类别时用条形图 |
| 时间趋势 | 折线图 | 时间点少于3个 |
| 部分与整体 | 饼图、环形图 | 类别过多或数值接近 |
| 数据分布 | 散点图、直方图 | 小数据集 |
| 多变量关系 | 气泡图 | 变量少于3个 |
| 地理分布 | 地图 | 无地理位置数据 |

## 常见图表详解

### 柱形图

#### 使用场景
```markdown
• 类别间比较（如：各季度销售额）
• 时间序列比较（如：2019-2024年收入）
• 排名展示（如：市场份额对比）
```

#### 设计要点
```markdown
✅ 正确做法：
• 柱子宽度 > 柱子间距
• 从0开始Y轴
• 使用水平标签
• 按大小排序（除非有固有顺序）

❌ 错误做法：
• 截断Y轴夸大数据
• 柱子过细或过密
• 旋转标签导致阅读困难
• 3D效果扭曲数据
```

#### 子类型选择
| 类型 | 适用场景 |
|------|----------|
| 簇状柱形图 | 多系列并排比较 |
| 堆叠柱形图 | 显示部分与整体关系 |
| 百分比堆叠柱形图 | 比较各部分占比 |

---

### 条形图

#### 使用场景
```markdown
• 类别名称较长
• 类别数量较多（>7个）
• 排名展示
• 强调第一名
```

#### 设计要点
```markdown
✅ 正确做法：
• 按数值大小排序
• 标签清晰可读
• 可添加数据标签
• 使用水平方向

❌ 错误做法：
• 按字母顺序排列（无意义）
• 标签截断
• 数值未从0开始
```

---

### 折线图

#### 使用场景
```markdown
• 展示数据随时间变化
• 识别趋势、周期、异常
• 多系列对比
• 预测未来走势
```

#### 设计要点
```markdown
✅ 正确做法：
• 从0开始Y轴（或明确标注截断）
• X轴时间间隔均匀
• 线条清晰（2-3pt）
• 合理的数据点数量（<20个）
• 添加数据点标记

❌ 错误做法：
• Y轴截断不断变化
• 线条过于密集
• 同时展示过多系列（>5条）
• 使用锯齿状线条
```

---

### 饼图

#### 使用场景
```markdown
• 展示部分与整体关系
• 类别数量少（≤5个）
• 不需要精确比较
• 强调某一个部分
```

#### 设计要点
```markdown
✅ 正确做法：
• 从12点钟方向开始
• 顺时针排列（从大到小）
• 使用标签直接标注
• 保留1位小数
• 切片不超过5个

❌ 错误做法：
• 切片超过7个
• 3D效果扭曲比例
• 分离过多切片
• 使用多个月牙形
```

#### 替代方案
```markdown
当饼图不合适时：
• 环形图：更现代，可中心放总计
• 百分比堆叠柱形图：更精确比较
• 表格：更精确展示数据
```

---

### 环形图

#### 使用场景
```markdown
• 百分比展示
• 多系列对比
• 空间感要求
• 中心可放汇总数据
```

---

### 散点图

#### 使用场景
```markdown
• 展示两个变量关系
• 识别数据分布模式
• 发现异常值
• 聚类分析
```

#### 设计要点
```markdown
✅ 正确做法：
• X轴和Y轴都从0开始
• 添加趋势线
• 适当大小的数据点
• 必要时添加标签

❌ 错误做法：
• 数据点过大重叠
• 使用过多颜色区分系列
• 缺少轴标签
```

---

### 气泡图

#### 使用场景
```markdown
• 三个变量关系（X、Y、气泡大小）
• 地理数据
• 资产配置分析
```

## 图表美化原则

### 简洁原则

| ✅ 简化 | ❌ 复杂化 |
|---------|-----------|
| 删除网格线 | 添加过多网格线 |
| 删除图例（直接标注） | 保留大图例 |
| 使用数据标签 | 依赖Y轴刻度 |
| 单一颜色或配色系列 | 五颜六色 |

### 对比原则

| ✅ 突出 | ❌ 平淡 |
|---------|--------|
| 用颜色强调重点 | 所有元素同一颜色 |
| 加大对比度 | 视觉模糊 |
| 重要数据突出 | 淹没在数据中 |

### 一致性原则

```markdown
同一PPT中的图表：
• 统一配色方案
• 统一字体
• 统一图例风格
• 统一数据标签格式
```

## 图表文案写作

### 标题撰写

```markdown
原则：标题即结论

❌ 弱标题：《2019-2024年销售额变化》
✅ 强标题：《5年间销售额增长320%，从1亿到4.2亿》

❌ 弱标题：《各产品线对比》
✅ 强标题：《A产品线领跑，占总营收的45%》
```

### 数据标签

```markdown
原则：清晰、简洁、必要

✅ 正确格式：
• 百分比：45%
• 金额：¥1.2亿
• 数字：1,234

❌ 错误格式：
• 精确到分：45.23%
• 带单位混用：1.2亿、12000万
• 小数点过多：1.234
```

### 注释说明

```markdown
必要注释：
• 数据来源
• 统计口径
• 时间范围
• 计算方法

位置：图表下方或备注页
```

## 常见图表错误

| 错误 | 问题 | 正确做法 |
|------|------|----------|
| Y轴截断 | 夸大数据差异 | 从0开始或明确标注 |
| 3D效果 | 扭曲数据比例 | 使用2D图表 |
| 饼图切片过多 | 难以阅读 | 合并为"其他"或用其他图表 |
| 网格线过密 | 干扰阅读 | 删除或减少网格线 |
| 图例位置不当 | 远离相关数据 | 图例在图表内或直接标注 |
| 颜色混乱 | 无意义配色 | 使用同一色系或逻辑配色 |
| 标签重叠 | 无法阅读 | 调整角度或使用数据标签 |
| 类型混用 | 比较逻辑错误 | 根据数据关系选择图表 |

## PptxGenJS图表代码

### 柱状图

```javascript
slide.addChart(pres.charts.BAR, {
  x: 1, y: 1, w: 8, h: 4,
  data: [
    { name: "系列1", labels: ["A", "B", "C"], values: [25, 35, 40] },
    { name: "系列2", labels: ["A", "B", "C"], values: [30, 25, 35] }
  ],
  chartColors: ["1A237E", "F57C00"]
});
```

### 折线图

```javascript
slide.addChart(pres.charts.LINE, {
  x: 1, y: 1, w: 8, h: 4,
  data: [ {
    name: "数据",
    labels: ["1月", "2月", "3月", "4月"],
    values: [10, 30, 25, 40]
  } ],
  chartColors: ["1A237E"],
  lineSize: 2,
  lineSmooth: true
});
```

### 饼图

```javascript
slide.addChart(pres.charts.PIE, {
  x: 1, y: 1, w: 5, h: 4,
  data: [ {
    name: "份额",
    labels: ["A", "B", "C", "其他"],
    values: [30, 45, 15, 10]
  } ],
  chartColors: ["1A237E", "3949AB", "F57C00", "E8EAF6"],
  showPercent: true,
  showValue: false
});
```

### 环形图

```javascript
slide.addChart(pres.charts.DOUGHNUT, {
  x: 1, y: 1, w: 5, h: 4,
  data: [ {
    name: "占比",
    labels: ["A", "B", "C"],
    values: [30, 45, 25]
  } ],
  chartColors: ["1A237E", "3949AB", "F57C00"],
  holeSize: 50
});
```
