# 常见错误与正确做法

## 内容设计错误

### ❌ 堆砌文字

```markdown
错误：
"各位领导、各位同事，大家好！今天我要向大家汇报的是关于我们公司2024年第一季度市场推广工作的情况..."

正确：
• 每页只说一件事
• 删除所有过渡语
• 标题即结论
• 控制在6行以内
```

### ❌ 标题模糊

```markdown
错误：
"工作汇报"、"情况说明"

正确：
"Q1营收增长25%，超额完成目标"
"用户流失率升至15%的3个原因"
```

## 视觉设计错误

### ❌ 颜色过多

```markdown
错误：超过5种颜色，每页配色不同

正确：
• 整个PPT只用3种颜色+黑白灰
• 7:2:1比例分配
• 全程一致
```

### ❌ 层次不清

```markdown
错误：
• 标题和正文一样大（14pt vs 16pt）
• 重点和次要同等突出

正确：
• 标题36pt+ vs 正文16pt（2倍以上差距）
• 重点加粗/变色
```

### ❌ 留白不当

```markdown
错误：
• 内容塞满页面
• 元素贴边或挤成一团

正确：
• 内容占60-70%
• 四周留白
• 相关元素紧凑
```

## 技术代码错误

### ❌ 使用#号

```javascript
// 错误 - 会损坏文件
color: "#FF0000"

// 正确
color: "FF0000"
```

### ❌ 使用async函数

```javascript
// 错误 - compile.js不会await
async function createSlide(pres, theme) { ... }

// 正确 - 必须是同步函数
function createSlide(pres, theme) { ... }
```

### ❌ 复用option对象

```javascript
// 错误 - PptxGenJS会修改对象
const opts = { fontSize: 18, color: "333333" };
slide.addText("Text1", opts);
slide.addText("Text2", opts);  // 可能出问题

// 正确 - 每次创建新对象
slide.addText("Text1", { fontSize: 18, color: "333333" });
slide.addText("Text2", { fontSize: 18, color: "333333" });
```

### ❌ 在阴影中编码opacity

```javascript
// 错误 - 会损坏文件
shadow: { color: "00000020" }

// 正确
shadow: { color: "000000", opacity: 0.12 }
```

## 对齐与版式错误

### ❌ 对齐混乱

```markdown
错误：
• 文字靠左又靠右
• 图片忽上又忽下

正确：
• 使用参考线
• 同一类元素严格对齐
• 建立网格系统
```

### ❌ 重复相同版式

```markdown
错误：每页内容页都用"标题+3个bullet"

正确：
• 第一页左图右文
• 第二页三栏卡片
• 第三页上下分栏
• 保持风格，变化版式
```

## 图表错误

### ❌ 选择错误的图表类型

```markdown
错误：用饼图展示趋势变化

正确：
• 比较 → 柱形图/条形图
• 趋势 → 折线图
• 占比 → 饼图/环形图
```

### ❌ 图表过于复杂

```markdown
错误：
• 3D效果扭曲数据
• 网格线过密
• 超过5个数据系列

正确：
• 2D图表
• 删除不必要的网格线
• ≤5个数据系列
```

---

## 动画错误

### ❌ 动画过多过花

```markdown
错误：
• 每页3种以上动画
• 弹跳、旋转、3D效果

正确：
• 最多2种动画（淡入、滑入）
• 轻柔、快速
• 服务于逻辑
```

## 正确做法总结

### 检查清单

**内容：**
- [ ] 每页只有一个核心观点
- [ ] 标题清晰有力
- [ ] 无废话和过渡语
- [ ] 逻辑清晰

**设计：**
- [ ] 配色不超过3种+黑白灰
- [ ] 字体对比≥2倍
- [ ] 所有元素对齐
- [ ] 留白适当

**技术：**
- [ ] 颜色6位HEX无#
- [ ] createSlide是同步函数
- [ ] 不复用option对象
- [ ] 无placeholder文本
